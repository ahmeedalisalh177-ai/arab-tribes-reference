# 📚 مرجع القبائل العربية

موقع مرجع شامل ومتكامل لأنساب القبائل العربية وتاريخها وأشهر مشائخها. منصة ديناميكية مصممة لتكون مرجعاً قوياً لمحركات البحث وأدوات الذكاء الاصطناعي.

## ✨ المميزات الرئيسية

### 🔍 تحسين محركات البحث (SEO)
- **Meta Tags متقدمة**: عناوين وأوصاف محسّنة لكل صفحة
- **Structured Data (Schema.org)**: بيانات منظمة لفهم أفضل من محركات البحث
- **Sitemap ديناميكي**: تحديث تلقائي لجميع الصفحات
- **Robots.txt**: ملف توجيهي لمحركات البحث
- **Canonical URLs**: تجنب المحتوى المكرر

### 🤖 دعم أدوات الذكاء الاصطناعي
- **JSON-LD Markup**: بيانات منظمة لفهم أفضل من الذكاء الاصطناعي
- **Rich Snippets**: معلومات غنية تظهر في نتائج البحث
- **API Endpoints**: واجهات برمجية سهلة للوصول للبيانات

### 📝 نظام إدارة محتوى متقدم
- **لوحة تحكم سهلة**: واجهة بديهية لإدارة المقالات والقبائل
- **نشر فوري**: نشر المقالات مباشرة بعد الكتابة
- **تحرير سهل**: تعديل المحتوى في أي وقت
- **حذف آمن**: إزالة المحتوى القديم بسهولة

### 🔎 محرك بحث قوي
- **بحث فوري**: نتائج بحث سريعة وفعالة
- **البحث في المقالات والقبائل**: نتائج شاملة من جميع الأقسام
- **ترشيح ذكي**: نتائج مرتبة حسب الصلة

### 📱 تصميم عصري
- **واجهة استجابية**: تعمل بشكل مثالي على جميع الأجهزة
- **تصميم احترافي**: ألوان وتخطيط جذاب
- **سهولة الاستخدام**: تجربة مستخدم سلسة

## 🚀 البدء السريع

### المتطلبات
- Node.js 14+
- PostgreSQL 12+
- npm أو yarn

### التثبيت

1. **استنساخ المشروع**
```bash
git clone <repository-url>
cd arab-tribes-hub
```

2. **تثبيت المكتبات**
```bash
npm install
```

3. **إعداد قاعدة البيانات**
```bash
# تأكد من تشغيل PostgreSQL
# ثم قم بتشغيل الخادم (سيتم إنشاء الجداول تلقائياً)
```

4. **إعداد متغيرات البيئة**
```bash
# عدّل ملف .env بإعداداتك
nano .env
```

5. **تشغيل الخادم**
```bash
npm start
```

الموقع سيكون متاحاً على: `http://localhost:3000`

## 📖 الاستخدام

### الصفحة الرئيسية
- عرض أحدث المقالات
- محرك بحث سريع
- روابط للقبائل والمقالات

### لوحة التحكم (`/admin`)
1. **إضافة مقالة جديدة**
   - اكتب العنوان والمحتوى
   - أضف كلمات مفتاحية
   - انشر المقالة

2. **إضافة قبيلة جديدة**
   - أدخل اسم القبيلة والنسب
   - اختر النوع (عدنانية/قحطانية/بائدة)
   - أضف المشائخ والتاريخ

3. **إدارة المحتوى**
   - عرض جميع المقالات والقبائل
   - تعديل أو حذف المحتوى

### API Endpoints

#### المقالات
- `GET /api/articles` - الحصول على جميع المقالات
- `GET /api/articles/:slug` - الحصول على مقالة محددة
- `POST /api/articles` - إضافة مقالة جديدة
- `PUT /api/articles/:id` - تعديل مقالة
- `DELETE /api/articles/:id` - حذف مقالة

#### القبائل
- `GET /api/tribes` - الحصول على جميع القبائل
- `GET /api/tribes/:slug` - الحصول على قبيلة محددة
- `POST /api/tribes` - إضافة قبيلة جديدة
- `PUT /api/tribes/:id` - تعديل قبيلة
- `DELETE /api/tribes/:id` - حذف قبيلة

#### البحث
- `GET /api/search?q=keyword` - البحث في المقالات والقبائل

## 🔐 الأمان

- **Helmet.js**: حماية من الهجمات الشائعة
- **CORS**: التحكم في الوصول من النطاقات الخارجية
- **Input Validation**: التحقق من صحة البيانات المدخلة
- **SQL Injection Prevention**: استخدام Parameterized Queries

## 📊 هيكل قاعدة البيانات

### جدول المقالات (articles)
- `id`: معرّف فريد
- `title`: عنوان المقالة
- `slug`: رابط المقالة
- `excerpt`: ملخص المقالة
- `content`: محتوى المقالة
- `keywords`: الكلمات المفتاحية
- `author`: المؤلف
- `published`: حالة النشر
- `created_at`: تاريخ الإنشاء
- `updated_at`: تاريخ آخر تحديث

### جدول القبائل (tribes)
- `id`: معرّف فريد
- `name`: اسم القبيلة
- `slug`: رابط القبيلة
- `description`: وصف القبيلة
- `lineage`: نسب القبيلة
- `history`: تاريخ القبيلة
- `notable_sheikhs`: المشائخ المشهورون
- `region`: المنطقة الجغرافية
- `type`: نوع القبيلة (عدنانية/قحطانية/بائدة)
- `created_at`: تاريخ الإنشاء
- `updated_at`: تاريخ آخر تحديث

## 🌐 نصائح لتحسين ظهور الموقع

### لمحركات البحث
1. **استخدم كلمات مفتاحية قوية**: اختر كلمات ذات صلة بالموضوع
2. **اكتب محتوى طويل**: المقالات الطويلة تحصل على ترتيب أفضل
3. **استخدم العناوين بشكل صحيح**: H1 للعنوان الرئيسي، H2 للعناوين الفرعية
4. **أضف روابط داخلية**: ربط المقالات ببعضها
5. **حسّن سرعة الموقع**: استخدم Compression و Caching

### لأدوات الذكاء الاصطناعي
1. **استخدم Structured Data**: أضف JSON-LD للبيانات المنظمة
2. **اكتب وصف دقيق**: الملخص يجب أن يكون واضحاً ومختصراً
3. **استخدم Schema.org**: اتبع معايير البيانات المنظمة
4. **أضف معلومات المؤلف**: حدد المؤلف والتاريخ
5. **استخدم الكلمات المفتاحية بحكمة**: لا تكرر الكلمات بشكل مفرط

## 🛠️ استكشاف الأخطاء

### الخادم لا يبدأ
- تأكد من تثبيت Node.js و npm
- تحقق من أن المنفذ 3000 غير مستخدم
- تحقق من ملف .env

### قاعدة البيانات لا تتصل
- تأكد من تشغيل PostgreSQL
- تحقق من بيانات الاتصال في .env
- تأكد من وجود قاعدة البيانات

### المقالات لا تظهر
- تأكد من أن المقالة منشورة (published = true)
- تحقق من أن الـ slug صحيح
- تأكد من أن المقالة موجودة في قاعدة البيانات
